How Mining Works
Step 01 — Data Input
Block index + previous hash + your data + miner address + nonce are concatenated into a single input string.
Step 02 — SHA-256 Hash
The string is fed to SHA-256 via the WebCrypto API. Output is always exactly 64 hexadecimal characters.
Step 03 — Check Target
If the hash starts with N leading zeros (= difficulty), we found a valid block! Otherwise nonce++ and repeat.
Step 04 — Chain & Immutability
Valid block is appended. Its hash becomes the prevHash of the next block. Tampering any block breaks all subsequent links.
💡 REAL SHA-256
This simulator uses the browser's native crypto.subtle.digest() — the same algorithm Bitcoin uses. Results are cryptographically valid, not simulated.
⬡ Blockchain — 1 Block(s)
⚠ CHAIN INTEGRITY VIOLATED — A block's data was tampered. Its hash no longer matches the prevHash stored in the next block. This is why blockchain data is effectively immutable: altering any block invalidates the entire chain from that point forward.
